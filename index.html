Why Hybrid Routing?

Not every prompt needs a trillion-parameter cloud model. Using the right tool for the job leads to better systems.

Feature Local LLM Cloud LLM HAR (Hybrid)
Cost Free (Your hardware) High (Usage-based) Optimized
Privacy Maximum (Data stays local) Variable (External API) Privacy-Aware
Speed Low Latency (Local) Higher Latency Balanced
Reliability Works Offline Fragile (Network/Outages) Automatic Fallback

Smart Routing

Categorizes prompts (chat, architecture, sensitive) to choose the absolute best model deterministically.

Strict Privacy

Mandatory deterministic redaction and strict LOCAL-only routing for sensitive payloads. Never leaks PII.

Rate Limiting

Built-in API key-aware rate limiting at the gateway to ensure production stability and cost control.

Explainable AI

Transparent Execution Trace that tells you exactly why a specific route was chosen and what providers were used.

Route Benchmarks

Professional evaluation tools to measure routing accuracy, safety, and latency distribution across datasets.

CI/CD Ready

Includes GitHub Actions for automated building, typechecking, and multi-stage secret scanning.

How HAR Decides

A multi-stage orchestrator evaluates every request with deterministic reasoning.

LOCAL

Simple tasks (greetings, rewriting) or sensitive data (PII/Secrets).

CLOUD

Complex reasoning, architecture design, deep coding tasks.

HYBRID

Multi-step. Clean logs locally -> Send to cloud for fix proposals.

Quick Start (5 Minutes)

HAR is designed to work out of the box with Ollama and your preferred cloud provider.

1

Install & Run Ollama

Download Ollama and pull a supported model.

ollama run llama3.2
2

Setup HAR

git clone https://github.com/FarhanS7/Hybrid-Router.git
cd Hybrid-Router
npm install
npm run build
3

Setup CLI & Environment

Link the CLI globally and copy the environment file.

npm link --workspace apps/cli
cp .env.example .env
4

Run & Verify

Start the background services and test a prompt in a new terminal.

# Terminal 1: Start services
npm run dev

# Terminal 2: Test
har "What is the capital of France?" --verbose

CLI Usage

har usage
# Simple prompt (usually routed to LOCAL)
~ $ har "What is the capital of France?"
# Complex prompt with verbose trace (usually routed to CLOUD)
~ $ har "Design a scalable notification system" --verbose
# Integration-friendly JSON output
~ $ har "Rewrite: 'hey how r u'" --json
# Check system health
~ $ har doctor
--verbose Shows routing reason
--json Structured data
--no-color Disable ANSI
